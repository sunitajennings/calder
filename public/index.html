"userless" mxconnect

Goal: create a version of MXconnect that does not rely on user creation to run. A “once and done” Connect that does not have continual knowledge of a user.

As a fintech developer, I want to verify an account and retrieve its account + routing number without creating persistent objects in the MX ecosystem.

Option 1: Using the MX Platform API 
GET  /institutions, filter for institution.supports_account_verification : true        
GET  /institutions/{institution_code}/credentials
POST /users
POST /users/{user_guid}/members with skip_aggregation : true
POST /users/{user_guid}/members/{member_guid}/verify
GET  /users/{user_guid}/members/{member_guid}/status
GET  /users/{user_guid}/members/{member_guid}/challenges (for account selection)
PUT  /users/{user_guid}/members/{member_guid}/resume
GET  /users/{user_guid}/members/{member_guid}/status
GET  /users/{user_guid}/members/{member_guid}/account_numbers
